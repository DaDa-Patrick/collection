動態背光燈光調整系統：提升觀看體驗的低成本解方（獨立完成）

成果影片： 系統展示影片

負責範圍

獨自完成此專案，包含整體系統設計、硬體選型與實作、軟體開發、演算法優化、測試與調試

欲解決問題：螢幕視覺效果不足，但現有解方成本過高

  1. 眼睛疲勞：長時間觀看螢幕時，螢幕與周圍環境的亮度差異會引起眼睛疲勞，影響視力健康。
  1. 沉浸式體驗不足：背光式螢幕無法提供環繞式的視覺效果，限制了觀看電影或遊戲時的沉浸感。
  1. 現有方案成本過高：雖然市面上已有販售類似系統，但單價過高無法普及。

專案總覽：提升螢幕使用體驗的低成本解方

設計並實現一個低成本的動態背光燈光調整系統，能夠根據螢幕內容即時調整背光燈光，提升觀看體驗並減少眼睛疲勞。此系統的價格僅為市售系統的 10% 以內，適合家庭娛樂和遊戲玩家。

技術與工具

  • 軟體：
    • Python：利用 Pillow 和 Quartz 庫進行螢幕截圖和顏色分析。
    • Arduino：為 ESP8266 編寫控制程式，接收顏色數據並控制 LED 燈光。
  • 硬體：
    • ESP8266 微控制器：用於接收來自電腦的顏色數據並控制 LED 燈條。
    • WS2812B LED 燈條：高亮度、多色的可編程 LED 燈條，實現背光燈光效果。

系統功能與流程

  1. 螢幕內容捕捉與顏色分析：
    • 使用 Python 程式定時截取螢幕邊緣的畫面，分析主要顏色。
    • 利用演算法計算螢幕四周的主色調，作為背光燈光的顏色參考。
  1. 燈光控制與同步：
    • 將分析出的顏色數據通過串行通信傳輸給 ESP8266 微控制器。
    • ESP8266 控制 WS2812B LED 燈條的顏色變化，使背光燈光與螢幕內容同步。

專案挑戰與解決方案

  1. 即時性和效能：
    • 問題原因：需要即時捕捉螢幕畫面並同步調整 LED 燈光，可能導致延遲和效能問題。
    • 解決方案：使用較低解析度的套件來進行高效的螢幕截取，並優化燈光變換算法，將更新頻率控制在每 16 毫秒，以達到近 60 FPS 的效果。
  1. 硬體整合：
    • 問題原因：ESP8266 與 WS2812B LED 燈條的兼容性和訊號同步不穩，影響燈光的準確呈現。
    • 解決方案：利用 FastLED 庫進行硬體驅動管理，並在 ESP8266 與電腦之間使用穩定的序列通訊協議，確保燈光訊號的傳輸可靠。
  1. 燈光同步的顏色準確性：
    • 問題原因：LED 燈條顏色與螢幕捕捉顏色存在偏差，無法準確還原螢幕畫面。
    • 解決方案：精確地使用 PIL 和 NumPy 對螢幕圖像進行縮放與取樣，將顏色數據平均化後傳遞給 LED 燈條，以確保顏色的相對準確性和一致性。

專案成果與特色

  • 即時同步效果：系統能根據電影、電視劇或遊戲內容動態調整背光燈光，提供環繞式的視覺效果，並且燈光反應速度低於 16 毫秒，達到即時同步的高效表現。
  • 低成本高效能：此專案以低成本實現了與市面上昂貴系統類似的功能，極大地降低了使用門檻，讓更多用戶能享受沉浸式體驗。
  • 實用性與可擴充性：系統設計簡單易用，安裝方便，並具備高度擴充性，未來可整合智慧家居，增強系統應用範疇。

ME_NU LLM 菜單推薦系統：基於大型語言模型的個性化餐點推薦（9人團隊合作完成）

成果頁面： ME_NU 介紹簡報

負責範圍

LangChain 與 ChatGPT API 的串接，進行 Prompt Engineering，提升對 LLM 的應用能力。

解決的問題

現代人在餐廳點餐時，經常面臨選擇困難、資訊不足或是無法確定菜品是否符合自己的口味，影響了用餐體驗和效率。

專案背景與簡介

ME_NU 是一款結合大型語言模型（LLM）的菜單推薦系統，旨在根據使用者的個人偏好和當前餐廳的菜單，提供個性化的餐點推薦，並整理相關的評價資訊，協助使用者快速決定餐點，提升用餐體驗。

技術與工具

  • LangChain：串接 ChatGPT API，實現 LLM 的功能。
  • ChatGPT API：用於自然語言生成和對話處理。
  • LINE Bot：提供多平台的互動介面，方便使用者使用。
  • Web App：建立網頁應用程式，提供友好的使用者介面。

專案特色

  • 個性化與情境化：根據使用者的偏好和情境，提供貼心的餐點推薦，提升用餐滿意度。
  • 創新性：結合 LLM 技術與餐飲服務，探索 AI 在生活應用中的可能性。
  • 實用性：解決了使用者在點餐時的痛點，節省時間，增進用餐體驗。

系統功能與流程

  1. 使用者偏好設定：
    • 使用者透過回答一系列問題，設定個人飲食偏好，如口味、飲食習慣、過敏原等。
  1. 菜單上傳與解析：
    • 使用者可上傳餐廳的菜單，系統會解析菜品資訊，為後續的推薦做準備。
  1. 個性化餐點推薦：
    • 系統結合使用者偏好、菜單和網路評價，透過 LLM 生成個性化的餐點推薦。
  1. 多角色對話互動：
    • 使用者可選擇不同的對話角色，系統會以相應的風格與使用者互動，增強體驗。

專案挑戰與解決方案

  1. 多角色情境對話生成：
    • 挑戰：需要實現不同角色風格的對話，如美食家、朋友或戀人，提供個性化的互動體驗。
    • 解決方案：利用 ChatGPT API，進行 Prompt Engineering，設計不同的提示語，讓 AI 生成符合角色特性的回應。
  1. 餐點資訊整合與推薦：
    • 挑戰：需要將使用者的偏好、餐廳菜單和網路評價進行整合，提供準確的推薦。
    • 解決方案：開發資料處理流程，將菜單與評價資料結合，並使用 LLM 進行分析和推薦。
  1. 系統效能與回應速度：
    • 挑戰：在使用者互動時，必須確保系統能夠快速回應，提供即時的推薦結果。
    • 解決方案：優化程式結構，使用高效的資料處理和緩存機制，降低系統延遲。

專案成果與特色

  • 個性化餐點推薦：系統能根據用戶的偏好和需求，即時提供符合口味的餐點推薦，並整合網路評論，讓用戶快速做出決策，減少選擇困難。
  • 多情境對話互動：利用不同風格的角色設定（如美食家、知心朋友、戀人），系統能針對用戶情感狀態和偏好進行情境化對話推薦，增強互動趣味性和沉浸感。
  • 跨平台支援與便捷操作：結合 LINE Bot 和 Web App，讓用戶無需安裝額外應用程式，透過掃描 QR code 或基於位置的菜單推薦，實現便捷的點餐體驗。
  • 即時評論整合與快速回應：系統自動整合 Google Maps 和其他餐廳評論，並將資訊濃縮成簡單的推薦描述，讓用戶在短時間內獲得有效資訊。

Your Sky Pylot：為觀星愛好者提供天文與地科資訊的爬蟲網站（3人團隊合作完成）

成果影片：網頁展示及說明影片

負責範圍

爬蟲開發、資料處理與分析、後端開發

解決的問題

許多天文愛好者在計劃出遊時，會因為天氣變化或缺乏即時天文資訊而影響體驗。而獲取以上資料需要拜訪多個網站進行許多複雜的資料填寫和搜尋。

專案背景與簡介

Your Sky Pylot 是一個專為喜愛出遊觀星的人設計的天文與地科資訊整合網站。使用者只需輸入地點與時間，即可獲得當日的天文預報、行星升落時間、月相變化、天氣預報以及空氣品質指標，幫助他們順利地規劃觀星活動。

技術與工具

  • 爬蟲技術：
    • Selenium：自動化瀏覽器，用於爬取 Google 搜尋趨勢及中央氣象局每日天文資料。
    • FeedParser：解析外國新聞的 RSS 資訊。
    • JSON Parser：解析空氣品質預報資料。
    • Google Trends API：取得國內新聞的熱門關鍵字及其相對搜尋熱度。
    • Google Translate API：翻譯外國新聞標題。
    • Beautiful Soup：解析 Time and Date 網站的行星資料。
  • 網站開發：
    • Django：用於搭建網站結構與後端功能。

系統功能與流程

  1. 主頁功能：
    • 地點與時間輸入：提供輸入框與日曆，讓使用者選擇出遊地點與時間。
    • 天文與地科新聞：顯示國內外熱門地科新聞，並提供外部連結。
    • 每日天文圖片：展示來自 NASA 的每日天文圖片，增添網站的趣味性。
  1. 結果頁面：
    • 天文預報：包括日出、日落時間，行星升落時間及亮度等資訊。
    • 月相與天文現象：顯示當天的月相及是否有特殊天文現象。
    • 天氣預報與空氣品質：提供當日的天氣預報，並以顏色標示空氣品質指標，方便使用者快速了解環境狀況。

專案挑戰與解決方案

  1. 首頁載入速度慢：
    • 問題原因：缺乏資料庫，每次載入首頁時都需要重新爬取新聞資訊，導致速度緩慢。
    • 解決方案：計劃建立資料庫，定期更新新聞資訊，減少每次載入時的爬取需求，加快首頁載入速度。
  1. 新聞篩選準確性不足：
    • 問題原因：部分爬取的搜尋結果與地科主題不符，影響使用者體驗。
    • 解決方案：優化爬蟲的關鍵字設定，增加篩選條件，確保爬取的新聞與地科相關。

專案成果與特色

  • 多功能天文與地科資訊整合：Your Sky Pylot 提供了全面的天文預報、行星升落時間、月相變化、天氣預報與空氣品質指標，幫助觀星愛好者更好地規劃出遊行程。
  • 跨平台資料整合：網站整合了來自多個來源的天文與氣象數據（如中央氣象局、NASA、Time and Date），確保資訊的廣泛性和準確性。
  • 即時動態更新：每日更新國內外的地科新聞及 NASA 天文圖片，保持資訊的新鮮度，增添網站的吸引力。

智慧情緒日記與分析平台（獨立完成）

平台連結

專案概述

此專案是一個智慧型的情緒分析與個人日記管理系統，利用人工智慧技術，透過使用者每日撰寫的日記內容進行深入的情緒分析，提供情緒辨識、認知行為療法（CBT）建議、以及個人化的鼓勵與回饋。

系統主要針對希望透過書寫日記來了解自我、改善心理健康狀態的使用者，結合 Streamlit 框架與 OpenAI 的 GPT-4-turbo 模型打造，並以 Google Cloud Platform（GCP）進行資料儲存與管理。

核心功能

1. 智慧情緒分析

  • 使用 GPT-4-turbo 針對日記內容進行情緒辨識，標註情緒類型並給予情緒強度百分比。
  • 提供詳細的認知行為療法（CBT）分析，包括情境、情緒、自動化思考、支持與不支持核心念頭的證據、替代性想法等。
  • 給予使用者專業且鼓勵性的建議，幫助使用者建立積極的心理狀態。

2. 個人化日記管理

  • 使用者可自由撰寫、編輯、刪除日記內容，所有內容將安全存儲於 GCP 的 Cloud Storage。

    • 採用市售低成本元件，提供大眾化的互動觸覺解決方案。
    • 具有較高的市場化與產品化潛力。

本專案成功打造出一套兼具實用性、互動性、低成本與舒適穿戴的腳部觸覺回饋系統，透過精細的控制與設計，達到高度的沉浸體驗，並經實驗驗證能有效提高使用者在虛擬環境中的互動真實感。

棒球資料系統：整合中華職棒資料的全端平台網頁（4人團隊合作完成）

負責範圍

  • 資料爬蟲、資料庫設計、後端 API 開發工作

欲解決問題

  1. 球隊與球員資訊分散難以查詢
    • 官方網站資訊分散且格式不一，使用者必須跳轉多個頁面才能取得完整資料
  1. 缺乏個人化追蹤與備註機制
    • 球迷無法記錄喜愛球員或自訂備註，互動性不足
  1. 資料更新流程繁瑣且容易錯誤
    • 手動蒐集、清理與格式化資料耗時費力，且易因人為疏漏導致錯誤

專案總覽

一句話總結：本專案旨在提供整合球隊、球員與賽程資料的互動式棒球資訊平台

關鍵特色：

  • Flask 後端結合 Firebase 驗證與 MySQL 資料庫，提供安全且可擴充的 API 服務
  • Next.js 前端動態顯示戰績、球隊與比賽資訊，並支援即時查詢
  • Google 登入與追蹤球員備註功能，大幅提升使用者互動性

技術與工具

  • 軟體開發
    • 語言／函式庫：Python（Flask、firebase_admin、mysql-connector）、TypeScript/React（Next.js）、Web 爬蟲：Firecrawl、Selenium、BeautifulSoup
    • 框架／平台：Flask、Next.js、Firebase Authentication、MySQL

系統功能與流程

  1. 資料爬取與整合
    • 使用 Firecrawl、Selenium 自動爬取官方網站的球員、球隊與賽程資料 → 清洗並格式化為 JSON
  1. 後端 API 與身份驗證
    • 以 Flask 架設 RESTful API，結合 Firebase Token 驗證機制 → 操作 MySQL 資料庫提供資料服務
  1. 前端介面與使用者互動
    • Next.js 前端動態渲染戰績、球隊列表及賽程資訊 → 使用者透過 Google 登入後，能追蹤球員並新增個人備註

專案挑戰與解決方案

  1. 第三方登入整合
    • 問題原因：需確保使用者身分安全且體驗流暢
    • 解決方案：導入 Firebase Admin 驗證 Token，撰寫 @check_auth 裝飾器保護 API 端點
  1. 大量球員資料解析與驗證
    • 問題原因：原始網頁資料結構不一致、欄位缺漏
    • 解決方案：以 ThreadPoolExecutor 提升爬取效率，搭配 BeautifulSoup 抓取欄位，並用 Pydantic 驗證及結構化資料
  1. 跨域請求限制
    • 問題原因：前後端部署於不同網域，瀏覽器預設阻擋跨域呼叫
    • 解決方案：在 Flask 啟用 CORS 設定，允許特定來源存取 API

專案成果與特色

  • 即時賽程與戰績查詢：前端透過 fetch 隨時取得最新戰績與比賽資料
  • 個人化追蹤與備註：使用者登入後可追蹤喜愛球員，並在球員頁面新增／查看個人備註
  • 自動化資料取得：爬蟲排程自動執行並整理官方資料，減少手動維護成本

校園安全守護站：互動式文章展示網站（獨立完成）

網頁連結

專案簡介

設計並實作一個以「校園安全」為主題的互動式前端網站，整合文章篩選、主題切換與 Markdown 動態載入功能。

網站能自動讀取 Markdown 文章，依照分類與日期動態呈現，並提供深淺色主題切換、滾動動畫與即時篩選排序。

整體採用純 HTML、CSS、JavaScript 實作，無需框架與伺服器即可離線執行，展示出我在前端互動設計與資料動態呈現上的整合能力。

負責範圍

獨立完成整體設計與開發，包含：

  • HTML 架構與語意化設計。
  • CSS 響應式排版與動畫效果。
  • JavaScript 互動邏輯、資料載入與使用者操作流程。

欲解決問題

  • 傳統文章網站缺乏互動性，難以快速篩選或閱讀特定主題。
  • 多數 CMS 架構複雜、需伺服器支援，對中小型專題或教育應用不友善。
  • 缺少以使用者體驗為核心的「閱讀導向」前端設計。

專案總覽：打造一站式的校園安全知識平台

本網站以「前端渲染 + Markdown 內容載入」為核心架構，能即時從本地資料夾讀取多篇文章，
並依據其分類（如「法律與規範」、「危機處理與防災」）進行篩選、排序與動畫展示。

網站同時支援「返回首頁」、「滾動進度條」與「回頂端」功能，提供流暢的單頁應用體驗（SPA-like）。

技術與工具

  • HTML5 / CSS3 / JavaScript（Vanilla JS）：純前端架構，無框架依賴。
  • marked.js：用於即時解析 Markdown 為 HTML。
  • LocalStorage：儲存使用者主題偏好。
  • IntersectionObserver API：實現滾動動畫與元素漸顯。
  • Fetch + Promise.all：同步載入多篇文章與封面。
  • CSS color-mix() / backdrop-filter：提供現代化漸層與光影效果。

系統功能與流程

  1. 文章載入與解析
    • 透過 Fetch 讀取 /articles/*.md 內的文章。
    • 使用 marked.js 將 Markdown 即時轉為 HTML，並自動擷取前 120 字生成摘要卡片。
  1. 分類與標籤篩選
    • 預設六大分類（法律與規範、校園安全案例、危機處理與防災、學生輔導與支持、制度與政策）。
    • 可多選標籤進行組合篩選，系統即時重新渲染列表。
  1. 日期排序切換
    • 支援「新 → 舊」與「舊 → 新」兩種排序方式，並於主頁與篩選列雙向同步。
  1. 深淺色主題切換
    • 圓形懸浮按鈕（FAB）切換模式，具日/月動畫過渡效果。
    • 使用 LocalStorage 記錄偏好並自動套用。
  1. 閱讀介面優化
    • 自動生成 TOC（目錄），可平滑滾動。
    • 若無封面圖自動降級為簡潔版版面。
    • 點擊標籤可回到清單頁進行對應篩選。
  1. 互動與動畫
    • 卡片滑入與滾動時具漸顯特效。
    • 滾動進度條顯示閱讀比例。
    • 提供「回頂端」與「回首頁」浮動按鈕。

專案挑戰與解決方案

  1. 無後端環境下的資料動態載入
    • 問題：純靜態架構無法即時存取資料。
    • 解法：以 Markdown + JSON 結合 fetch() 實現前端動態渲染。
  1. 多分類與排序的同步控制
    • 問題：多層級篩選與排序容易衝突。
    • 解法：以全域狀態物件管理 chips 與排序狀態，確保 UI 同步。
  1. 深淺色主題動畫過渡
    • 問題：切換瞬間閃爍不自然。
    • 解法：使用 CSS 變數與 color-mix() 平滑過渡，搭配延遲動畫呈現光暈。

專案成果與特色

  • 完全前端執行：可直接在本地開啟，無需伺服器。
  • 即時互動與動畫：文章載入、滾動與篩選皆有過渡效果。
  • 良好延展性：可作為任何主題的多文章展示框架。
  • 現代化設計語言：採半透明介面、圓角卡片、漸層光暈與主題切換動畫。
